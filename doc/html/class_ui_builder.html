Detailed Description

A class for dynamically visualizing a SQRL identity within the application's MainWindow GUI.

UiBuilder, upon being provided with an IdentityModel object, will dynamically render the identity's block structure by creating all the necessary UI controls and stylesheets "on the fly" and inserting them into the MainWindow GUI.

◆ setEnableUnauthenticatedChanges()

void UiBuilder::setEnableUnauthenticatedChanges ( bool  enable,
bool  rebuild = true 
)

Sets an internal flag that signals whether unauthenticated changes to the identity should be enabled or disabled.

Any settings and values within a SQRL identitiy are being protected against unauthorized manipulation by applying the AES-GCM authenticated encryption. With this method, the plain text within an identity is included with the encryption of the identity keys as "additional data". Therefore, if any information within an identity changes, the decryption of the keys will fail.

"Unauthenticated changes" are changes to the identity which don't get cryptographically "reauthenticated". This means, any well behaving client will reject such "rogue" identities.

If enable is true, unauthenticated changes are being enabled, if it is set to false, they are being disabled.

If rebuild is set to true, any change is immediately reflected within the visual representation of the identity. For example, if unauthenticated changes are being disabled and rebuild is true, buttons for editing and moving identity blocks and items are removed from the GUI.

◆ showGetBlockTypeDialog()

bool UiBuilder::showGetBlockTypeDialog ( QString *  result,
bool  allowEdit = false 
)
static

Displays a dialog, asking the user to select or enter a block type.

The dialog provides a list of existing block types to choose from (meaning block types that do have a block definition file present). If allowEdit is true, the user can enter a block type that does not exist, allowing for the creation of new block type definitions. If allowEdit is false, the user can only choose from the list of existing block types.

If the dialog was not cancelled and the user chose or entered a block type, true is returned and the block type is put into result. Otherwise, false is returned.

Detailed Description

A class for managing multiple loaded identities in a tabbed interface.

TabManager holds a list of IdentityTab objects which are being displayed using a QTabWidget. It provides methods for adding, removing and managing tabs/identities.

◆ setEnableUnauthenticatedChanges()

void TabManager::setEnableUnauthenticatedChanges ( bool  enable,
bool  rebuild = true 
)

Sets an internal flag that signals whether unauthenticated changes to the identity tabs should be enabled or disabled.

Any settings and values within a SQRL identitiy are being protected against unauthorized manipulation by applying the AES-GCM authenticated encryption. With this method, the plain text within an identity is included with the encryption of the identity keys as "additional data". Therefore, if any information within an identity changes, the decryption of the keys will fail.

"Unauthenticated changes" are changes to the identity which don't get cryptographically "reauthenticated". This means, any well behaving client will reject such "rogue" identities.

If enable is true, unauthenticated changes are being enabled, if it is set to false, they are being disabled.

If rebuild is set to true, any change is immediately reflected within the visual representation of the identity. For example, if unauthenticated changes are being disabled and rebuild is true, buttons for editing and moving identity blocks and items are removed from the GUI.

◆ updateCurrentTabText()

void TabManager::updateCurrentTabText ( )

Updates the currently active tab's text and tooltip using information provided by the tab itself.

If file information is present for the current tab's identity, the file name and path will be used for setting the tab's text and tooltip. Otherwise, a name of "Untitled" will be used.

An asteriks ("*") will be shown as a suffix to the tab text if the tab is dirty (has unsaved changes).

It is advisable to call this method after an operation that either influences the "dirty state" of a tab, or the file information.
